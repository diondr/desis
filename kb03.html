Penjadwalan Processor

Processor atau CPU pada komputer merupakan tempat dimana proses-proses dari program dijalankan. Agar penggunaan CPU dapat berjalan dengan maksimal perlu adanya pengaturan waktu dalam menjalankan program. Pengaturan itu merupakan tugas sistem operasi agar proses berjalan sepanjang waktu dan memperkecil waktu tunggu (idle). Oleh karena itu perlu adanya penjadwalan proses-proses tersebut.

Penjadwalan processor adalah suatu proses pengaturan atau penjadwalan proses-proses yang ada di dalam komputer. Dimana proses-proses tersebut berjalan dalam pola yang disebut "Siklus Burst". Penjadwalan processor secara garis besar dibagi menjadi 2, yaitu penjadwalan preemptive dan penjadwalan non preemptive.

Keberhasilan dari penjadwalan processor tergantung dari beberapa bagian processor. Pengeksekusian dari proses tersebut terdiri atas siklus “CPU ekskusi” dan “I/O Wait”. Proses hanya akan bolak-balik dari dua keadaan (state) tersebut, inilah yang disebut Siklus Burst CPU-I/O. Pengeksekusian proses dimulai dengan Burst CPU, setelah itu diikuti oleh Burst I/O, kemudian Burst CPU lagi lalu Burst I/O lagi, dan seterusnya dilakukan secara bergiliran. Burst CPU terakhir akan berakhir dengan permintaan sistem untuk mengakhiri pengeksekusian melalui Burst I/O lagi. Kejadian siklus Burst dapat dilihat pada gambar 2.

Dari gambar di atas dapat kita lihat bahwa waktu Burst CPU yang lama jarang terjadi karena kebanyakan proses akan melakukan antara output ke layar atau file, maupun meminta input dari user atau file sehingga situasi dimana proses bekerja dengan memory dan register dalam jangka waktu lama tidak terlalu banyak ditemukan.

Penjadwalan Preemptive

Penjadwalan processor mungkin akan dijalankan ketika proses dalam keadaan:

  1. Berubah dari running state ke waiting state.
  2. Berubah dari running state ke ready state.
  3. Berubah dari waiting state ke ready state.
  4. Dihentikan.

Penjadwalan preemptive adalah kemampuan sistem operasi untuk memberhentikan sementara proses yang sedang berjalan untuk memberi ruang kepada proses yang prioritasnya lebih tinggi. Penjadwalan ini bisa saja termasuk penjadwalan proses atau I/O. Penjadwalan preemptive memungkinkan sistem untuk lebih bisa menjamin bahwa setiap proses mendapat sebuah waktu khusus untuk beroperasi. Dan juga membuat sistem lebih cepat merespon terhadap event dari luar (contohnya seperti ada data yang masuk) yang membutuhkan reaksi cepat dari satu atau beberapa proses. Membuat penjadwalan yang Preemptive mempunyai keuntungan yaitu sistem lebih responsif daripada sistem yang memakai penjadwalan non preemptive.

Penjadwalan nomor (2) dan (3) bersifat preemptive sedangkan (1) dan (4) bersifat non preemptive. Pada Linux penjadwalan yang digunakan bersifat preemptive. Bahkan beberapa penjadwalan sistem operasi pada Linux tersebut, memiliki kemampuan preemptive terhadap system call-nya (preemptible kernel).

Penjadwalan Non Preemptive

Penjadwalan non preemptive merupakan salah satu jenis penjadwalan dimana sistem operasi tidak pernah melakukan pergantian dari proses yang sedang berjalan ke proses yang lain. Dengan kata lain, proses yang sedang berjalan tidak bisa diganggu keberadaannya.

Penjadwalan non preemptive terjadi ketika proses hanya:

  1. Berubah dari running state ke waiting state.
  2. Dihentikan.

Hal tersebut berarti CPU menjaga proses sampai proses itu pindah ke waiting state ataupun dihentikan (proses tidak diganggu). Tentu saja Linux tidak menggunakan metode penjadwalan ini.

Rangkuman

Penjadwalan processor atau CPU adalah pemilihan proses dari antrian ready untuk dapat dieksekusi. Penjadwalan processor merupakan konsep dari multiprogramming, dimana CPU digunakan secara bergantian untuk proses yang berbeda. Suatu proses terdiri dari dua siklus yaitu Burst I/O dan Burst CPU yang dilakukan bergantian hingga proses selesai. Penjadwalan processor mungkin dijalankan ketika proses:

  1. Berubah dari running state ke waiting state.
  2. Berubah dari running state ke ready state.
  3. Berubah dari waiting state ke ready state.
  4. Dihentikan.

Proses 2 dan 3 adalah proses pada Penjadwalan Preemptive, dimana proses tersebut boleh di-interrupt, sedangkan 1 dan 4 adalah proses pada Penjadwalan Non Preemptive, dimana proses tidak bisa di-interrupt. Penjadwalan yang digunakan pada Linux yaitu penjadwalan yang bersifat preemptive.
